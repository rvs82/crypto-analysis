Техническое задание на создание сайта для криптовалютного трейдинга:
1. Общее описание: Сайт должен предоставлять рекомендации по торговле криптовалютами (LDO/USDT, AVAX/USDT, XLM/USDT, HBAR/USDT, BAT/USDT, AAVE/USDT) с использованием максимального количества технических индикаторов и искусственного интеллекта для анализа данных и принятия решений.
2. Функционал: Отображение текущих рыночных настроений в двух горизонтальных блоках: "Лонги" (зелёный) и "Шорты" (красный) с целыми процентами (не более 100%). Таблица с данными по каждой паре: название пары, рекомендация (направление, вход, стоп-лосс, тейк-профит, уверенность, соотношение риск/прибыль), текущая сделка, количество закрытых сделок, стопов, профитов, общая прибыль и убыток в USDT. Открытие сделок только при уверенности не ниже 50% с высокой стабильностью тренда после обработки всех индикаторов искусственным интеллектом. Звуковое уведомление при открытии новой сделки (включение/выключение через чекбокс). Под таблицей: список всех используемых индикаторов и фильтров, принципы принятия решений об открытии сделок, параметры активных сделок (неизменные до закрытия).
3. Дизайн: Шрифт в таблице на 1 пиксель меньше стандартного (15px). Слово "Лонг" выделять зелёным цветом, "Шорт" — красным. Вместо "Стоп" использовать "С", вместо "Профит" — "П". "Лонги" и "Шорты" в шапке сайта в виде двух горизонтальных блоков: зелёный для "Лонги", красный для "Шорты".
4. Технические требования: Использовать все возможные технические индикаторы для анализа (25 и более), включая трендовые, импульсные, волатильные, объёмные и уровневые. Применять искусственный интеллект (например, линейную регрессию) для прогнозирования цены и повышения точности. Учитывать ложные сигналы: фильтровать перекупленность/перепроданность, слабые тренды, высокую волатильность. Обновление данных каждые 5 секунд через WebSocket с Binance. Стабильность: сделки открываются только при подтверждённом сильном тренде с минимальными колебаниями и после обработки всех индикаторов ИИ.
5. Тестирование сделок: Перед запуском протестировать код локально с использованием mock-данных для проверки корректности открытия сделок при уверенности 50% и выше. Убедиться, что сделки открываются только после полного анализа всех индикаторов с применением ИИ. Проверить закрытие сделок по стоп-лоссу и тейк-профиту, а также точность расчёта прибыли и убытков. Логи должны отображать обновление цен, открытие и закрытие сделок без ошибок. После деплоя проверить работоспособность на реальных данных Binance, убедиться в отсутствии багов и корректности прогнозов.
6. Дополнительно: Сайт должен быть надёжным, без багов, с точными прогнозами, минимизирующими убытки. Все данные (индикаторы, принципы, сделки, техзадание) должны быть доступны пользователю на главной странице.
